## 2. nexus-java — Java 后端

### 目录导航

```
nexus-java/src/main/java/org/ibo/nexusjava/
├── NexusJavaApplication.java          ← 启动类, @MapperScan + @SpringBootApplication
├── common/                             ← 全局基础设施
├── config/                             ← Spring Bean 装配
├── interceptor/                        ← 请求拦截
├── messaging/                          ← RocketMQ 消息消费
├── modules/                            ← 业务领域
└── resources/                          ← YAML 配置
```

### 2.1 启动类

**`NexusJavaApplication.java`**
- 唯一的 `main` 入口
- `@MapperScan("org.ibo.nexusjava.modules.*.mapper")` 扫描所有模块的 MyBatis Mapper
- 排除 `RocketMQAutoConfiguration` 防止自动配置报错

### 2.2 common/ — 通用层

| 文件 | 作用 | 关键设计 |
|------|------|----------|
| `Result.java` | 统一 API 响应壳 | 泛型类 `Result<T>`, 静态工厂 `Result.success(data)` / `Result.error(code, msg)` |
| `ErrorCode.java` | 错误码枚举 | 按模块分段: 200=成功, 4000=参数错误, 1100~=认证, 1200~=用户, 1300~=订阅, 1400~=通知, 1500~=审批, 1600~=审计, 5000=系统错误 |
| `BusinessException.java` | 业务异常 | 继承 RuntimeException, 携带 ErrorCode + 自定义消息, GlobalExceptionHandler 统一捕获 |
| `GlobalExceptionHandler.java` | 全局异常处理器 | `@RestControllerAdvice`, 三层 catch: BusinessException → MethodArgumentNotValidException → Exception |
| `JwtUtil.java` | JWT 工具 | `@Component`, HMAC-SHA256, `generateToken(userId, username, role)` 返回带 Claims 的 Token, `parseToken(token)` 校验并提取 |
| `RequestIdFilter.java` | 请求ID过滤器 | 继承 `OncePerRequestFilter`, 优先读取请求头 `X-Request-Id`, 否则生成 UUID, 写入 MDC(日志) + 响应头 |

### 2.3 config/ — 配置层

| 文件 | 作用 | 关键细节 |
|------|------|----------|
| `WebConfig.java` | Web 全局配置 | 三件事合在一个类: (1) 注册 `RequestIdFilter` 在 `/api/*` 路径, 最高优先级 (2) CORS 允许 localhost:5173/3000 (3) 注册 `JwtAuthenticationInterceptor`, 排除 `/api/java/auth/**` |
| `MybatisPlusConfig.java` | MyBatis-Plus 插件 | 仅注册分页插件 `PaginationInnerInterceptor(DbType.MYSQL)`, 无其他配置 |
| `RedisConfig.java` | Redis 序列化配置 | 自定义 `RedisTemplate<String, Object>`, Key 用 String 序列化, Value 用 Jackson JSON 序列化 |

### 2.4 interceptor/ — 拦截器

| 文件 | 作用 | 调用链 |
|------|------|--------|
| `JwtAuthenticationInterceptor.java` | JWT 校验拦截器 | `preHandle`: `Bearer xxx` → 解析 JWT → 写入 UserContext + request.setAttribute → 401 则返回 JSON |
| `UserContext.java` | ThreadLocal 用户上下文 | 静态方法: `setUserId(id)`, `getUserId()`, `clear()`, 每个请求结束后由拦截器 afterCompletion 清理 |

> **为什么用 ThreadLocal?** 避免在每个 Service 方法签名中传递 userId, 防止水平越权 (用户不能伪造别人的 userId 来删别人数据)。

### 2.5 messaging/ — RocketMQ 消费者

#### DTO (4 个消息体)

| 文件 | 字段 |
|------|------|
| `TaskTriggerMessage.java` | taskId, subscriptionId, triggerTime |
| `ContentParsedMessage.java` | taskId, subscriptionId, sourcePlatform, sourceUrl, title, summary, contentBody, author, publishedAt, contentHash, vectorId |
| `NotificationMessage.java` | userId, type, title, content, relatedId |
| `DeadLetterMessage.java` | originalTopic, originalMessage, failureReason, retryCount |

#### Consumer (4 个消费者)

| 文件 | Topic | 做了什么 |
|------|-------|----------|
| `TaskTriggerConsumer.java` | nexus-task-trigger | 更新 `subscription.last_run_at` |
| `ContentParsedConsumer.java` | nexus-content-parsed | 收到 Python 解析结果 → INSERT content |
| `NotificationConsumer.java` | nexus-notification | INSERT notification |
| `DeadLetterConsumer.java` | nexus-dead-letter | INSERT audit_log (死信记录) |

> 全部加 `@ConditionalOnProperty(name="rocketmq.enabled", havingValue="true")`, 确保本地没有 RocketMQ 也不报错。

### 2.6 modules/ — 业务模块

每个业务模块遵循相同的分层结构:

```
modules/{domain}/
├── controller/    # REST 端点
├── service/       # 业务逻辑接口 + impl
├── entity/        # MyBatis-Plus 实体 → 数据库表
├── mapper/        # BaseMapper 接口
├── vo/            # 返回给前端的视图对象 (脱敏/聚合)
├── dto/           # 前端传来的请求体 (校验注解)
```

#### 2.6.1 auth — 认证模块

| 文件 | 路径 | 关键逻辑 |
|------|------|----------|
| `AuthController.java` | POST /register, POST /login | 唯一不在拦截器白名单外的模块 |
| `RegisterDTO.java` | — | @NotBlank username, @Size(min=6) password, @Email email |
| `LoginDTO.java` | — | username + password |
| `LoginVO.java` | — | token, tokenType, expiresIn, user(Map) |
| `AuthServiceImpl.java` | — | register: BCrypt 加密 → INSERT user(role='user'); login: BCrypt 比对 → 生成 JWT |

#### 2.6.2 subscription — 订阅管理 (核心模块)

| 文件 | 关键逻辑 |
|------|----------|
| `SubscriptionController.java` | 完整 CRUD + `POST /{id}/trigger` |
| `SubscriptionServiceImpl.java` | **trigger() 方法**: 生成 UUID taskId → Redis 分布式锁 `SET NX PX 300000` → 尝试 RocketMQ 发送 → 失败则 HTTP POST `http://localhost:8000/api/python/agent/tasks?subscription_id={id}` |
| `SubscriptionVO.java` | keywords/sourcePlatforms 从 JSON 字符串反序列化为 List\<String\> |

> **trigger 的核心**: Java 不直接执行采集，而是通知 Python Agent。这是异步解耦的关键设计。

#### 2.6.3 content — 内容模块

| 文件 | 关键逻辑 |
|------|----------|
| `ContentServiceImpl.java` | `listBySubscription`: LambdaQueryWrapper 条件筛选 + Page 分页; `getById`: 查询后关联 subscription 表拿订阅名 |
| `ContentVO.java` | subscriptionName (JOIN 得来), relatedContents (JSON→List) |

#### 2.6.4 notification — 通知模块

`NotificationServiceImpl.java`: 四个方法 — 分页列表 (按 userId + 时间倒序), 未读计数, 单条已读, 全部已读。所有方法用 `UserContext.getUserId()` 确保只看自己的。

#### 2.6.5 approval — 审批模块

`ApprovalTicketServiceImpl.java`:
- `listPending`: status=0, 分页
- `approve`: 校验 status=0 → 设 status=1 + approvedBy + approvedAt
- `reject`: 同上但 status=2

#### 2.6.6 audit — 审计模块

纯只读模块, 分页查 audit_log 表。

#### 2.6.7 system — 系统模块

| 端点 | 数据来源 |
|------|----------|
| /metrics | 4 个 `selectCount` (user/subscription/content/approval where status=0) |
| /charts | 全量查 content → Java 内存 GROUP BY sourcePlatform + 按小时聚合 fetchedAt |

## 3. nexus-python — Python AI 中台

### 目录导航

```
nexus-python/app/
├── main.py              ← FastAPI 实例创建 + lifespan + router 注册
├── api/                  ← REST 层
├── agents/               ← AI Agent (LangGraph)
├── core/                 ← 基础设施
├── memory/               ← 向量存储
├── messaging/            ← 消息消费
├── mcp_servers/          ← MCP 子进程
├── models/               ← DB 模型
└── services/             ← 业务门面
```

### 3.1 main.py — 应用入口

```python
app = FastAPI(title="NexusAI Python 中台")
# lifespan:
#   startup  → MCPPool 注册 + Milvus 连接 + RocketMQ Consumer 启动
#   shutdown → Consumer 关闭 + MCP 清理 + Milvus 断开 + DB engine dispose
# routers:
#   agent, content, subscription, notification
# 内置:
#   GET /health, GET /api/python/mcp/servers
```

### 3.2 core/ — 配置与依赖

| 文件 | 作用 | 要点 |
|------|------|------|
| `config.py` | 全局配置中心 | Pydantic Settings v2, 6 个子配置类: LLM, Milvus, RocketMQ, Playwright, DB, Redis。支持 `.env` + 环境变量。`get_settings()` 是单例。 |
| `dependencies.py` | FastAPI 依赖注入工厂 | `get_db_session()` 返回 `AsyncSession` (aiomysql), `get_redis_client()` 返回 Redis 连接。所有资源懒初始化。 |
| `exceptions.py` | 自定义异常 | `NexusAIException` → `AgentExecutionError`, `MCPConnectionError`, `MilvusOperationError`, `RiskInterruption` |

### 3.3 api/ — REST 路由

| 文件 | 说明 |
|------|------|
| `agent.py` | 核心入口: POST /tasks 触发 Scout Agent (查 subscription → 解析 keywords/platforms → `asyncio.create_task(run_scout_task(...))`) |
| `content.py` | 列表/详情/语义搜索 |
| `subscription.py` | 列表/详情/触发 |
| `notification.py` | 列表/已读 |

### 3.4 agents/ — AI Agent 核心

| 文件 | 作用 | 架构 |
|------|------|------|
| `state.py` | 共享状态定义 | TypedDict: taskId, subscriptionId, keywords, sourcePlatforms, rawItems, parsedContents, duplicateIds, storedIds, status, error |
| `scout_agent.py` | **主采集流水线** | 4 步管道: (1) RSS fetch (2) 元数据检测→网页刮取 (3) 非中文→LLM翻译 (4) hash去重→存储→通知。另有 LangGraph 版本 (fetch_sources→parse_content→check_duplicate→store_content→notify) 保留备用。 |
| `supervisor.py` | LangGraph 状态图 | 构建 StateGraph, 注册 5 个节点 + 条件边, 使用 `_sync_node` 包装异步节点 |

> **为什么有两套?** `run_scout_task()` 是简化直调版 (本地开发用，不依赖 LangGraph 线程隔离)。`supervisor.py` 是完整 Agent 版 (生产用)。

### 3.5 memory/ — 向量数据库

| 文件 | 作用 |
|------|------|
| `milvus_client.py` | Milvus 单例客户端: 连接/建Collection(IVF_FLAT+COSINE)/插入向量/相似度检索(`search_similar`)。`_check_connected()` 确保本地无 Milvus 时优雅降级(返回空结果,不报错)。 |

### 3.6 messaging/ — 消息消费

| 文件 | 作用 |
|------|------|
| `consumer.py` | RocketMQ PushConsumer, 订阅 `nexus-task-trigger`, 收到消息→新线程 event loop→调用 `run_scout_task()`。ImportError 时跳过。 |

### 3.7 mcp_servers/ — MCP 工具进程

| 文件 | 作用 | 工具 |
|------|------|------|
| `config.py` | ServerConfig 定义 | name, command, args, transport, idle_ttl |
| `registry.py` | 注册表 | 注册 4 个服务器: email, web, rss, api |
| `client_wrapper.py` | 单连接管理 | stdio 子进程→`ClientSession`→`call_tool()` |
| `pool.py` | 连接池 | 懒加载, 健康检查(30s), GC回收(60s) |
| `nexus_mcp_rss.py` | RSS 工具 | `fetch_rss(url)` → feedparser |
| `nexus_mcp_web.py` | 浏览器工具 | `scrape_page(url)` → Playwright |
| `nexus_mcp_api.py` | HTTP 工具 | `call_api(endpoint)` → httpx |
| `nexus_mcp_emails.py` | 邮件工具 | `parse_email`, `send_email` → SMTP |

> **MCP 工作原理**: 每个工具作为独立 Python 子进程运行, 通过 stdin/stdout 的 JSON-RPC 与主进程通信。主进程通过 `ClientWrapper` 管理每个子进程的生命周期。

### 3.8 models/ — 数据库模型 (SQLAlchemy 2.0)

| 文件 | 表 | 关键字段类型 |
|------|-----|-------------|
| `base.py` | — | `DeclarativeBase` |
| `user.py` | user | BigInt PK, username unique |
| `subscription.py` | subscription | JSON 列: keywords, source_platforms, trigger_conditions |
| `content.py` | content | CHAR(36) PK, content_hash unique, LongText 正文 |
| `approval_ticket.py` | approval_ticket | JSON context, TINYINT status |
| `notification.py` | notification | TEXT content, composite index (user_id, is_read) |

## 8. 设计模式与约定

### 分层架构

```
Controller (接收请求, 参数校验)
    ↓
Service (业务逻辑, 事务边界 @Transactional)
    ↓
Mapper (MyBatis-Plus BaseMapper, LambdaQueryWrapper 拼条件)
    ↓
Entity (POJO → 数据库表一行)
```

- **Controller 层**: 仅做路由 + 参数接收，不写业务逻辑
- **Service 层**: 所有增删改查 + 权限校验 (UserContext.getUserId)
- **Mapper 层**: 继承 BaseMapper, 复杂查询用 LambdaQueryWrapper
- **VO vs Entity**: Entity 直连数据库, VO 面向前端 (脱敏/聚合/反序列化)

### 命名约定

| 层次 | 命名 | 示例 |
|------|------|------|
| Controller | `{Domain}Controller` | `SubscriptionController` |
| Service 接口 | `{Domain}Service` | `SubscriptionService` |
| Service 实现 | `{Domain}ServiceImpl` | `SubscriptionServiceImpl` |
| Mapper | `{Domain}Mapper` | `SubscriptionMapper` |
| Entity | `{Domain}` | `Subscription` |
| VO | `{Domain}VO` | `SubscriptionVO` |
| DTO | `{Action}DTO` / `{Domain}QueryDTO` | `RegisterDTO`, `SubscriptionQueryDTO` |

### 安全模式

```
请求 → Gateway JwtGlobalFilter (校验Token→注入Header)
     → Java JwtAuthenticationInterceptor (解析→UserContext ThreadLocal)
     → Controller
     → Service (UserContext.getUserId() 使用当前用户, 不从 request 参数取)
```

### 异步消息模式

```
Java trigger() → RocketMQ (nexus-task-trigger)
                         ↓ (不可用时)
              HTTP → Python Agent → feedparser → 去重 → MySQL
```

## 修改指南

| 想做什么 | 改哪里 |
